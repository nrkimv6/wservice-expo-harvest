## 요구사항 정리

1. `/app` 대신 `/`를 실제 앱 진입점으로 올리고, `/`가 왜 존재하는지 모호한 상태를 정리한다.
2. 한 페이지에 길게 이어진 정보 배치를 하단 네비게이션 중심 구조로 바꿔 상황별 중요도를 분리한다.
3. `/app` 상단 소개 카드(`Map First`, `Search + Filter`, `Separate State`)는 상시 노출 대신 첫 실행 1회 모달로 이동한다.

## 현재 코드 관찰

- `src/routes/+page.svelte`: 마케팅 랜딩 성격의 소개 페이지이며, 실제 사용 흐름은 `/app` 링크로 넘긴다.
- `src/routes/app/+page.svelte`: 알림 배너, 소개 카드, 박람회 선택, 요약 카드, 지도, 리스트가 한 화면에 모두 쌓여 있다.
- `src/lib/stores/farmState.ts`: 전시회별 로컬 상태와 마지막 선택 박람회만 저장하며, 온보딩 노출 여부는 저장하지 않는다.
- `src/service-worker.ts`: 앱 셸과 네비게이션 fallback이 `/app`을 포함한 현재 라우팅을 전제로 한다.
- `README.md`: 메인 동작 설명과 오프라인 진입 설명이 `/app` 기준으로 작성되어 있다.

## 구현 순서 제안

### Phase 1: 루트 라우트 정리

1. [ ] **실사용 엔트리를 `/`로 통합한다** — 랜딩과 앱 진입을 분리하지 않고 모바일 앱의 기본 URL을 하나로 고정
   - [ ] `src/routes/+page.svelte`: 현재 랜딩 페이지 대신 실제 파밍 화면을 렌더링하는 주 라우트로 재구성한다.
   - [ ] `src/routes/app/+page.svelte`: 기존 `/app` 직접 진입은 `/`로 보내는 호환 리다이렉트 페이지 또는 load redirect로 단순화한다.

2. [ ] **라우트 정리의 부수 영향 범위를 같이 맞춘다** — 오프라인 fallback과 문서가 새 엔트리와 모순되지 않게 정리
   - [ ] `src/service-worker.ts`: 앱 셸 캐시 목록과 navigate fallback 우선순위를 `/` 중심으로 재정렬한다.
   - [ ] `README.md`: 앱 진입 경로와 오프라인 설명을 `/` 기준 문구로 갱신한다.

### Phase 2: 하단 네비게이션 정보 구조

3. [ ] **한 화면에 몰린 섹션을 탭 단위로 분리한다** — 홈/지도/리스트/저장됨처럼 목적이 다른 화면을 하단 네비로 전환
   - [ ] `src/routes/+page.svelte`: 현재 단일 세로 스크롤 레이아웃을 탭 상태 기반 조건부 렌더링 구조로 바꾼다.
   - [ ] `src/app.css`: 하단 고정 네비게이션과 safe-area 대응 여백을 추가해 모바일에서 가려지지 않도록 조정한다.

4. [ ] **탭별 우선 정보를 재배치한다** — 항상 봐야 하는 정보와 필요할 때만 보는 정보를 분리
   - [ ] `src/routes/+page.svelte`: 알림 배너와 현재 박람회 핵심 요약은 기본 탭에 두고, 지도/리스트/저장 상태는 개별 탭으로 분리한다.
   - [ ] `src/lib/components/LootFeed.svelte`: 탭 전환 구조에서 재사용 가능하도록 리스트 헤더/빈 상태 문구가 현재 화면 맥락과 맞는지 정리한다.

5. [ ] **박람회 선택과 저장 상태가 새 구조에서도 자연스럽게 이어지게 한다** — 탭을 바꿔도 맥락이 끊기지 않게 유지
   - [ ] `src/routes/+page.svelte`: 박람회 선택, 선택된 부스 상세 시트, 찜/완료 카운트를 탭 구조 안에서 일관되게 유지한다.
   - [ ] `src/lib/components/ExhibitionMap.svelte`: 지도 탭 단독 노출 시 시각 밀도와 상단 정보량이 과하지 않도록 컨테이너 문구/배치를 재검토한다.

### Phase 3: 첫 실행 온보딩 모달

6. [ ] **소개 카드를 첫 실행 1회 모달로 옮긴다** — 상시 노출 대신 처음 진입할 때만 핵심 사용법을 안내
   - [ ] `src/routes/+page.svelte`: 기존 상단 소개 카드 내용을 모달 UI로 옮기고, 첫 방문 시 자동 오픈되도록 연결한다.
   - [ ] `src/app.css`: 모달 오버레이, 바텀 고정 네비와의 겹침, 스크롤 잠금 스타일이 자연스럽게 보이도록 필요한 유틸을 보강한다.

7. [ ] **온보딩 노출 여부를 로컬에 저장한다** — 이후 재방문에서는 자동으로 다시 뜨지 않게 처리
   - [ ] `src/lib/stores/farmState.ts`: 온보딩 완료 여부를 읽고 저장하는 localStorage helper를 추가한다.
   - [ ] `src/routes/+page.svelte`: 모달 닫기와 “다시 보기” 트리거를 연결해 최초 1회 + 수동 재진입 흐름을 지원한다.

### Phase 4: 회귀 점검 및 문구 정리

8. [ ] **새 구조 기준으로 카피와 메타 정보를 정리한다** — 루트 진입과 탭 구조가 문서/메타에 반영되도록 정리
   - [ ] `src/routes/+page.svelte`: 페이지 title/description과 화면 카피를 새 엔트리 구조에 맞게 조정한다.
   - [ ] `README.md`: 현재 포함 범위 설명에서 랜딩 표현을 줄이고, 하단 네비 및 첫 실행 모달 구조를 반영한다.

9. [ ] **정적 검증으로 라우트/타입 회귀를 확인한다** — 문서와 코드 변경이 SvelteKit 빌드 기준과 충돌하지 않는지 검증
   - [ ] `package.json`: 기존 `check` 스크립트를 사용해 `npm run check` 기준으로 오류 없이 통과하는지 확인한다.
   - [ ] `src/service-worker.ts`: `/app` 잔존 참조가 필요한 호환 범위인지 최종 점검하고, 불필요한 fallback 분기를 제거한다.

## 기대 결과

- 사용자는 `/`에서 바로 앱을 시작하고, `/app`은 과거 링크 호환용 진입점으로만 남는다.
- 첫 화면은 길게 늘어진 종합 페이지가 아니라, 하단 네비게이션으로 목적별 화면을 빠르게 오갈 수 있는 구조가 된다.
- 소개 카드는 첫 진입에만 보여 주고, 이후에는 화면을 가리지 않으면서 필요 시 다시 열 수 있다.

## 리스크 및 확인 포인트

- `/app`을 바로 제거하면 기존 북마크나 설치형 PWA 진입이 깨질 수 있으므로 최소한 한 차례는 리다이렉트 호환이 필요하다.
- 하단 네비게이션을 넣으면 기존 상세 바텀시트와 safe-area 여백이 겹칠 수 있어 모바일 실기기 간격 점검이 필요하다.
- 첫 실행 모달 상태 키를 추가할 때 기존 로컬 저장 스키마와 충돌하지 않도록 별도 키로 분리해야 한다.
