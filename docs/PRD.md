# PRD: 박람회 이벤트 파밍 최적화 웹앱 (MVP)

## 1. 제품 개요

- 목적: 박람회 관람객이 부스별 이벤트를 놓치지 않고 최단 동선으로 참여해 파밍 효율을 극대화하도록 돕는 모바일 우선 웹앱
- 핵심 가치: 오프라인 환경에서도 안정적으로 동작하고, 빠르게 탐색 가능하며, 상태 관리가 직관적이어야 한다
- 타겟 유저: 제품 구매나 비즈니스 네트워킹보다 사은품 수령과 이벤트 참여를 우선하는 실속파 관람객

## 2. 핵심 유저 스토리

- 사용자는 시간대별 선착순 이벤트를 시간순으로 정렬해서 확인할 수 있다
- 사용자는 이벤트 참여 조건을 미리 확인하고 필요한 앱을 준비할 수 있다
- 사용자는 전시장 도면 위에서 관심 부스의 위치를 확인하고 이동 동선을 계획할 수 있다
- 사용자는 파밍을 완료한 부스를 완료 처리해서 지도와 리스트에서 구분할 수 있다
- 사용자는 특정 부스의 필수 해시태그나 개인 팁을 텍스트 메모로 남길 수 있다

## 3. 주요 기능 요구사항

### 3.1 글로벌 알림 배너

- 형태: 상단 고정 롤링 배너 또는 티커
- 목적: "10분 뒤 A부스 룰렛 시작" 같은 시간 임박 이벤트를 빠르게 노출
- 구현 방향: 서버/디바이스 푸시 대신 실시간 데이터 브로드캐스팅 활용

### 3.2 지도 및 부스 안내도

- 층별/관별 이미지 맵 제공
- 핀치 줌 및 드래그 스크롤 지원
- 이벤트가 있는 부스만 지도 위에서 강조
- 부스 핀 상태:
  - 기본
  - 관심
  - 완료(흐리게 또는 시각 분리)
- 핀 탭 시 바텀 시트로 부스명, 이벤트 내용, 경품 즉시 노출

### 3.3 파밍 리스트 및 검색

- 텍스트 검색 지원
- 다중 필터링 지원
- 검색 대상:
  - 부스명
  - 경품명
  - 이벤트 종류
- 정렬 기준:
  - 시간 의존 이벤트 최상단
- 필터 조건:
  - 시간제한
  - SNS 업로드
  - 단순 팔로우
  - 앱 설치
  - 회원가입

### 3.4 상세 정보 및 로컬 메모

- 부스별 상세 미션/경품/위치 제공
- 자유 메모 텍스트 입력
- 메모 저장:
  - `localStorage`
  - 부스 ID를 key로 사용
  - debounce 적용
- 상태 컨트롤:
  - 관심 토글
  - 완료 토글

## 4. 시스템 및 기술 제약

- 프레임워크: SvelteKit + Supabase
- 공용 데이터:
  - Supabase PostgreSQL
  - Realtime 적용
- 개인 데이터:
  - `localStorage`
  - 메모, 찜, 완료 상태를 우선 로컬 저장
- 오프라인 지원:
  - PWA 기반
  - 최초 로드시 전체 부스 JSON과 지도 이미지를 캐시
  - 네트워크 불안정 상황에서도 리스트 탐색과 메모가 동작해야 함
